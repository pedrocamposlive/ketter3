# UI Release Blueprint — Ketter 3.0

Regra Base
Cada item do state-ui-manus.md deve ser executado com:
- Edição de theme.css  
- Edição de tokens.js  
- Edição de App.css  
- Edição de componentes React  
- Edição de Layout.jsx  
- Edição das páginas (Dashboard, Transfers, Audit, Health, Settings)  
- Limpeza de CSS morto  
- Aplicação dos tokens Manus  
- Ajustes de responsividade  
- Atualização do state  
- Commit automático  

Ciclo
Para cada tarefa:
1. Ler objetivo  
2. Procurar todos os pontos relacionados no código  
3. Aplicar as mudanças  
4. Reforçar tokens Manus  
5. Reforçar responsividade  
6. Atualizar documentação  
7. Commitar  

Estrutura
As tarefas se dividem em:

1. Theme Tokens  
   - spacing  
   - radius  
   - shadows  
   - transitions  
   - tipografia  
   - grid  
   - cinzas  
   - cores  

2. Layout Global  
   - ui-container  
   - page-shell  
   - spacing global  
   - breakpoints Manus  
   - header Manus  

3. Dashboard  
   - KPIs  
   - Throughput Chart  
   - Donut Chart  
   - Alerts Panel  
   - Status Grid  
   - hierarchy spacing  

4. Transfers Page  
   - layout  
   - cards  
   - details  
   - severidade  
   - spacing  

5. Audit Page  
   - timeline  
   - marcadores  
   - legibilidade  
   - blocos  
   - spacing vertical  

6. Health Page  
   - métricas  
   - heatmap/status  
   - hierarchy  

7. Settings Page  
   - grid  
   - fieldsets  
   - toggles  
   - enterprise layout  

8. Final Polish  
   - limpeza global  
   - normalização  
   - consistência total  

Resultado Final
A UI deve:
- Refletir o design Manus  
- Ser responsiva  
- Ser uniforme  
- Ser enterprise  
- Ser pronta para apresentação executiva e auditoria
