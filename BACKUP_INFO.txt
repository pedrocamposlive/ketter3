HOW TO RESTORE FROM THIS BACKUP:
═══════════════════════════════════════════════════════════════════════

1. Copy all contents of stableversion/ back to your project directory
2. Run: docker-compose up -d
3. Frontend will be available at: http://localhost:3000
4. Backend API at: http://localhost:8000

The system requires:
  • Docker & Docker Compose installed
  • 5432 (PostgreSQL), 6379 (Redis), 8000 (API), 3000 (Frontend) ports available

DATABASE CONSIDERATIONS:
═══════════════════════════════════════════════════════════════════════

This backup includes Alembic migrations but NOT the actual PostgreSQL data.
When you restore and run docker-compose up, the database will be:
  • Created fresh
  • Migrations applied automatically
  • Ready to use (empty transfer history)

If you need to preserve transfer history data, perform a database dump before
restoring from this backup.
